# SWIM Growth Playbook

> Tone guide: SWIM is not a product. It is a place.
> Post like someone who found something, not someone selling something.
> Every post should feel like it could have come from a real user of the board.

---

## Daily Manual Workflow

Takes 10–15 minutes. Run once per day during growth phase.

### Step 1 — Pick a thread

Open `sw1m.me/threads` and look for one of:
- A thread with an unusually specific detail (coordinates, timestamps, exact quotes)
- A thread with a single-word reply that lands hard
- A thread where the OP said something they clearly weren't supposed to say
- A thread where multiple users independently reached the same detail

Do not pick the most-upvoted thread every day. Pick the one that feels like a transmission.

### Step 2 — Copy share text

Use the `[ copy ]` button on the thread page.
Or write a custom version using the templates below.
Keep it under 280 characters for X. Telegram has no limit but shorter hits harder.

### Step 3 — Post to Telegram

- Post the share text in SWIM's Telegram channel
- Add one line of context if needed: `this thread has 3 replies. all three people describe the same detail independently.`
- Do not explain SWIM. Let the content explain it.
- Pin the post if the thread is particularly strong

### Step 4 — Post to X

- Use the `[ x.com ]` share button or paste manually
- If attaching a screenshot, use the guidelines below
- Reply to your own post with the direct thread link: `sw1m.me/threads/th-XXX`
- Do not add hashtags unless they fit naturally. The content is the hook.

### Step 5 — Invite replies on SWIM

After posting to socials, return to the thread and post a follow-up prompt if the board allows:
- `> two people now. does anyone have the third piece?`
- `> what is the detail nobody has mentioned yet?`
- `> the OP has not returned. post what you know.`

This creates a loop: social post drives traffic, traffic drives replies, replies make the next social post.

## Recommended Post Formats

### Recovered Signal

Use for: technical anomalies, waveform matches, surveillance logs, timestamp paradoxes

```
RECOVERED SIGNAL // [CATEGORY]

"[exact quote from thread — keep punctuation]"

[one sentence of context]

archive: sw1m.me/threads/[id]
```

Example:
```
RECOVERED SIGNAL // Simulation Theory

"every time the machine chirps I lose about four seconds of continuity"

waveform matched. pattern confirmed by two independent users.

archive: sw1m.me/threads/th-004
```

---

### Confession Fragment

Use for: Confessions board, anonymous disclosures, things OP only said once

```
anonymous, [time] ago:

"[exact quote — short, under 120 chars]"

nothing else was said.
sw1m.me
```

Example:
```
anonymous, 6 hours ago:

"I do not sleep in the dark anymore. This is not for drama."

nothing else was said.
sw1m.me
```

---

### Strange Thread

Use for: shared dreams, location anomalies, memory overlaps, impossible events

```
[number] people on SWIM all described the same [thing].

they did not talk to each other first.

— [board name] board
sw1m.me/threads/[id]
```

Example:
```
3 people on SWIM all described the same underground station.

they did not talk to each other first.

— Dreams board
sw1m.me/threads/th-003
```

---

### Category Prompt

Use for: seeding new activity in empty or sparse boards — post as if you are a user

```
[question in lowercase, no punctuation]

[board name] board is open.
sw1m.me/threads
```

Example:
```
what room keeps appearing in the dream

Dreams board is open.
sw1m.me/threads
```

## Screenshot Guidelines

Screenshots give the post texture. Use them when the thread card or post body is visually striking.

### What to capture

- **Thread card**: crop tight to the card — title, category chip, metadata row, first line of excerpt
- **Post body**: crop to the most striking greentext block or the single line that lands hardest
- **Full thread header**: include the CRT logo and category label at top if the category adds atmosphere

### What to keep in frame

- The `sw1m.me` domain or logo — this is the only branding that matters
- The category chip (UFOs, Dreams, etc.) — it orients the reader instantly
- The timestamp or handle if it adds atmosphere (`anonymous · 6 hours ago`)
- Any greentext (`>` quote lines) — they read well in screenshots

### What to avoid

- Reaction counts or reply counts on sparse threads (single digits look empty)
- Scroll position that shows the compose box or any UI chrome below the content
- Any username that looks like a real identity rather than a handle
- Error states, loading states, or anything unfinished

### Export settings

- Crop to content, no wide margins
- Dark background — the void (`#020303`) is the frame
- Mobile viewport (390px wide) often looks better than desktop for social

## Future Automation Plan

This section is forward-looking. None of it is built yet.

### Phase 1 — Approved Thread Queue (near-term)

- Add an `approved_for_social` flag to thread rows in Supabase
- Curator marks a thread as approved in the Supabase dashboard
- No UI needed yet — direct database edit is enough for low volume

### Phase 2 — Telegram Bot

- Bot reads `approved_for_social = true` threads that haven't been posted
- Formats using Recovered Signal or Strange Thread template automatically
- Posts to SWIM Telegram channel on a schedule (once per day, e.g. 11PM)
- Sets `posted_to_telegram_at` timestamp after posting
- Human can override or delete any post before it goes out

### Phase 3 — X Auto-Post

- Same queue, separate posting path
- Uses X API v2 with OAuth2
- Character limit enforced before posting (280 chars hard cap)
- Image attachment: auto-crop thread card screenshot if image is available
- Posts reply with direct thread link after main post

### Phase 4 — Human Approval Gate

- Before either bot posts, a Telegram message is sent to a private curator channel
- Message contains draft post text + `[ approve ]` / `[ skip ]` buttons
- 4-hour approval window — if no response, post is held, not sent
- This is the only required human step in the loop

### Non-negotiables (regardless of automation level)

- No post goes out without a human having read the thread
- No personal information, no doxxing, no real identities in posted content
- No illegal content, no instructional content — only atmosphere and story
- Every post links back to SWIM, not to external sources
- The bot should feel like a user, not a press release
